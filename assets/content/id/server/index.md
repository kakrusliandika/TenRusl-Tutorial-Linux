# 🖥️ Server

## Ringkasan
Bagian server adalah sumber utama bahasa Inggris untuk dasar server Linux, pintu depan layanan yang menghadap web, bidang kendali hosting, dan platform aplikasi yang di-host sendiri. Tujuannya bukan hanya untuk menginstal perangkat lunak. Tujuannya adalah untuk membangun server yang tetap dapat dijelaskan, dipulihkan, dan diverifikasi dari shell.

## Untuk Siapa Bagian Ini
- Admin Linux membangun atau menstandardisasi baseline server.
- Operator VPS yang memerlukan jalur berulang dari OS baru ke layanan produksi.
- Tim membandingkan administrasi Linux langsung, panel hosting, proxy terbalik, dan alur kerja platform yang di-host sendiri.

## Prasyarat
- VM lab, VPS, atau server fisik yang dapat Anda bangun kembali jika diperlukan.
- Konsol, hypervisor, atau akses penyelamatan untuk pemulihan jaringan dan SSH.
- Keakraban dasar dengan paket, layanan, penyimpanan, DNS, dan akses shell.

## Jalur Pembelajaran
1. Mulailah dengan [📦 OS Dasar](./base-os/index.md) untuk menetapkan identitas, SSH, firewall, pembaruan, dan penyimpanan yang dapat diprediksi.
2. Gunakan [🌐 Proxy Terbalik](./reverse-proxy/index.md) ketika host harus menyajikan layanan HTTP atau HTTPS dengan aman.
3. Pilih [🌐 Panel Hosting](./hosting-panels/index.md) jika server akan menjalankan situs web atau tumpukan aplikasi yang dikelola panel.
4. Pilih [🚀 PaaS](./paas/index.md) jika server akan menghosting alur penerapan aplikasi layanan mandiri.
5. Sebelum penghentian produksi, kunjungi kembali [🛡️ Keamanan](../security/index.md) dan [💾 Cadangan](../backup/index.md).

## Yang Harus Dibaca Terlebih Dahulu

# 🚀 PaaS

## Overview
The PaaS section covers self-hosted application platforms that run on top of Linux server infrastructure. The focus is operational rather than marketing-driven: repository trust, container visibility, domain routing, backup scope, service maps, and controlled recovery when the platform or workload misbehaves.

## Who This Section Is For
- Operators building self-hosted application delivery on their own Linux servers.
- Teams evaluating platform-driven deploy workflows against direct shell administration.
- Admins who need a host-aware way to audit what the platform changes.

## Prerequisites
- A server with CPU, RAM, and storage sized for container workloads.
- SSH access, DNS control, and outbound network access.
- Basic familiarity with containers, Git providers, and reverse proxy behavior.

## Learning Path
1. Start with [🌐 Coolify](./coolify/index.md).
2. Read the conceptual pages before connecting repositories or exposing domains.
3. Implement backup and restore paths before production onboarding.
4. Pair the module with [🛡️ Security](../../security/index.md) and [💾 Backup](../../backup/index.md).

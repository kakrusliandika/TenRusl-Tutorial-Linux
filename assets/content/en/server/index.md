# 🖥️ Server

    ## Overview
    The server section is the English source of truth for Linux server baselines, web-facing service front doors, hosting control planes, and self-hosted application platforms. The goal is not just to install software. The goal is to build servers that stay explainable, recoverable, and verifiable from the shell.

    ## Who This Section Is For
    - Linux admins building or standardizing server baselines.
    - VPS operators who need a repeatable path from fresh OS to production service.
    - Teams comparing direct Linux administration, hosting panels, reverse proxies, and self-hosted platform workflows.

    ## Prerequisites
    - A lab VM, VPS, or physical server you can rebuild if needed.
    - Console, hypervisor, or rescue access for network and SSH recovery.
    - Basic familiarity with packages, services, storage, DNS, and shell access.

    ## Learning Path
    1. Start with [📦 Base OS](./base-os/index.md) to establish identity, SSH, firewall, updates, and predictable storage.
2. Use [🌐 Reverse Proxy](./reverse-proxy/index.md) when the host must present HTTP or HTTPS services safely.
3. Choose [🌐 Hosting Panels](./hosting-panels/index.md) if the server will run panel-managed websites or application stacks.
4. Choose [🚀 PaaS](./paas/index.md) if the server will host self-service application deployment flows.
5. Before production cutover, revisit [🛡️ Security](../security/index.md) and [💾 Backup](../backup/index.md).
